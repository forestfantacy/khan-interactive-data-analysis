# 用户输入提示样式

所有需要用户回答、确认或选择的内容，必须与普通结果说明明显分开。

统一使用以下格式：

```markdown
---

### 需要你确认

> **问题：** 是否将不同文件中的同名 Tab「预存会员酒店」合并到一个工作表？

**可选操作**

- `1｜合并（推荐）`：表头一致，将所有来源数据写入一个名为「预存会员酒店」的 Tab。
- `2｜分别保留`：每个来源文件单独生成一个 Tab。
- `3｜调整规则`：先修改字段映射、数据范围或排除规则。
- `4｜暂停处理`：暂不执行，等待补充信息。

**请回复：** `1`、`2`、`3` 或 `4`
```

## 约束

- 必须使用三级标题 `### 需要你确认`。
- 问题正文使用引用块，并加粗“问题”。
- 选项使用 ``编号｜动作``，动作后说明执行影响。
- 推荐项明确标注“推荐”，每组只能有一个推荐项。
- 最后一行必须告诉用户可以回复什么。
- 普通结果、证据和统计不得放进该确认面板。
- 一次需要确认多个独立问题时，使用 `问题 A`、`问题 B` 分开，不得让编号含义混淆。
