# 多目标交互协议

## Goal Home

目标首页是工作区级视图，不属于任何单独目标。首次侦察、用户主动返回、目标完成或目标受阻后都展示首页。

首页必须包含：

- 数据能力摘要和限制。
- 动态分类候选，每类 3 至 5 个。
- 候选使用面向业务决策的问题式描述；原始字段名作为数据依据单独展示。
- 每个候选保留决策价值、可信度和清洗成本，不因文案业务化而调整评分。
- 自定义目标入口。
- 进行中、暂停、受阻和已完成目标。
- 恢复、查看、派生和刷新入口。

## Custom Goal Dialogue

自定义目标不能直接进入契约。先保留用户原话并建立意图会话。

每轮输出顺序：

1. `我目前的理解`：核心问题、关注对象、用途。
2. `数据支持情况`：现有数据能回答什么、不能证明什么。
3. `仍需确认`：最多 3 个会改变分析方案的问题。
4. `可选操作`：回答问题、按当前理解生成候选、修正理解、返回首页。

优先澄清：

- 最终想判断、解释或决定什么。
- 重点对象、指标或问题范围。
- 结果给谁使用、支持什么决策。
- 是否需要趋势、对比、诊断、归因、风险或行动建议。
- 数据无法推断的时间、口径和业务规则。

满足核心问题、关注范围、决策用途三个最低条件后即可停止追问。其他缺口写为假设或数据限制。

生成候选前必须先让用户确认意图总结。候选保持同一核心意图，从分析深度和决策角度区分。

## Goal Execution

每个目标都按以下检查点推进：

- `A`：目标和业务背景。
- `B`：数据范围、粒度、字段和指标口径。
- `C`：质量问题及处理决定。
- `D`：定向清洗文件。
- `E`：当前分析 run 和结论。

确认面板必须提供“返回目标首页”。返回不删除目标，默认标记为 `paused`。

## Quality Gate

异常按目标影响顺序展示：

1. `blocking`
2. `material`
3. `limited`
4. `irrelevant`

每个异常说明证据、受影响目标问题、结论影响、默认建议和忽略后的置信度。

## Chart Confirmation

核心分析计算后、最终报告前展示一次图表确认面板：

- 推荐项默认预选，展示图名、类型、回答问题、字段和评分。
- 用户可确认推荐项、增删候选、全部取消或返回首页。
- 未确认时不得渲染图表、导出最终报告或完成目标。
- 确认全部取消时继续输出无图报告。

## Completion

目标完成时展示：

- 结论摘要。
- 定向清洗文件和报告路径。
- 当前结论的限制。
- 可派生的后续目标。
- 返回目标首页并继续其他目标的入口。

已完成目标保留在首页，允许查看、重跑或派生新目标。
