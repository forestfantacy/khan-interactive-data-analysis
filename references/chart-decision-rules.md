# 图表判定规则

## 1. 用户模式

- `自动判定`：按评分决定是否出图，默认模式。
- `不出图`：不生成图表，但仍可使用必要的数据表格。
- `需要图表`：优先生成有分析价值的图表；数据质量不支持时仍允许 0 张。

## 2. 评分

| 条件 | 分值 |
| --- | ---: |
| 直接回答核心分析问题 | +3 |
| 揭示趋势、结构、对比、分布或关系 | +2 |
| 仅用文字不容易看清 | +2 |
| 直接支撑行动建议 | +2 |
| 数据量和类别数量适合阅读 | +1 |
| 与已有图表重复 | -2 |
| 数据质量或口径尚未确认 | -3 |
| 只能起装饰作用 | -3 |

判定：

- `>= 6`：必须出图
- `3-5`：可选
- `< 3`：不出图

## 3. 数量上限

- 简要：最多 2 张
- 标准：最多 5 张
- 深入：最多 10 张

上限不是配额。没有合格图表时允许 0 张。

## 4. 典型映射

| 分析问题 | 图表 |
| --- | --- |
| 时间变化 | 折线图 |
| 对象高低 | 排序柱状图 |
| 结构构成 | 堆叠柱状图 |
| Top 贡献 | 帕累托图 |
| 分布与异常 | 箱线图 |
| 指标关系 | 散点图 |

图表必须服务于已确认的分析目标，并在报告中解释业务含义。
